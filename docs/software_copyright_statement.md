# NongY 项目说明（软著材料）

## 开发意义
- 面向智慧农业和家庭植物养护场景，整合环境传感、植物识别、AI 助手和远程控制，降低种植门槛并提高管理效率。
- 通过移动端实时监控与可视化，帮助用户及时发现环境异常，减少因信息滞后导致的损失。
- 借助图像分析和多模态交互，提供轻量级的植物健康诊断与指导，提升用户体验和决策质量。

## 需求分析
- 业务需求：支持多终端用户登录；监测土壤湿度、温湿度、光照等环境数据；对植物图像进行识别与健康分析；提供环境控制（灌溉、风扇、照明等）和历史数据查询。
- 功能需求：传感数据采集与展示、设备控制命令下发、图像上传与识别、AI 对话咨询、通知提醒、账号与资料管理、扩展的学习/娱乐模块。
- 非功能需求：移动端流畅交互；本地缓存和数据库存储；支持多分辨率布局；接口可配置，便于对接不同传感器网关和云服务。

## 功能与模块
1) 账号与用户中心：登录/注册、个人资料维护（ProfileActivity）、设备与通知设置（SystemSettingsFragment 等）。
2) 设备与环境监测：传感器数据读取与可视化（SensorAdapter、MonitorFragment 及各设备子模块），支持土壤湿度、光照、温湿度等；设备状态显示。
3) 植物识别与图像分析：图像上传/拍照后调用分析接口（ImageAnalysisApi、PlantAnalyzeFragment），返回植物名称、健康建议等。
4) AI 助手与聊天：AI 问答与多轮对话（AiChatActivity、ChatAdapter/ChatMessage），辅助诊断、提供养护建议。
5) 控制与自动化：对接 ESP32/硬件设备的控制接口（ESP32Api、DeviceSensorApi），支持泵、风扇、照明等控制命令下发。
6) 数据与存储：本地数据库与数据模型（MyDbHelper 及 domain 包），用于历史记录、用户数据、Mini Program 数据等。
7) Mini Program 与扩展模块：菜单/课堂/备忘录等轻应用容器（MiniProgramContainer、Memo/MiniProgram 相关适配器），方便扩展业务小模块。
8) 教育与娱乐：内置 2048、俄罗斯方块、贪吃蛇、马里奥、塔防等游戏片段，提供互动体验；音乐、日历、地图等通用工具模块。
9) UI 与交互：丰富的界面布局与动画（res/layout、res/anim/drawable 等），支持底部导航、多 Tab 模块化入口。

## 使用说明
- 运行环境：Android Studio（Gradle 构建，见根目录 gradle/ 与 settings.gradle.kts），需配置可访问的后端接口与硬件网关地址（参考 `AppConfig`/`HttpUrlConnectConfig`）。
- 启动与登录：安装 APK 后通过登录/注册进入主界面；首次使用可在“个人/系统设置”中完成设备绑定和通知权限配置。
- 设备监测：在“监控/环境”模块查看实时传感数据；异常值会以状态或颜色提示，支持查看历史列表。
- 植物分析：在“植物/识别”模块拍照或选择图片，提交后获得识别结果与养护建议。
- AI 助手：在 AI 聊天界面提问种植、养护或系统操作相关问题，支持持续对话。
- 控制操作：在设备控制界面对泵、风扇、灯光等进行开关/模式切换，命令通过接口下发至硬件。
- 扩展模块：通过 Mini Program 菜单进入课堂、备忘录、工具和娱乐模块；相关数据会存储在本地数据库，部分支持网络同步。
- 构建与发布：在 Android Studio 中选择目标设备或打包 Release，确保签名与后端/硬件地址配置正确后再分发。

## 软著填报用环境信息
- 开发的硬件环境：x86-64 台式机/笔记本（Intel/AMD），16GB 以上内存，500GB SSD（示例）。
- 运行的硬件环境：Android 智能手机/平板（ARMv7/ARMv8，建议 Android 8.0+，摄像头/麦克风/网络正常）；可选外接 ESP32 传感器网关与传感器设备。
- 开发该软件的操作系统：Windows 10/11（示例，可用 macOS 13+/Ubuntu 20.04+）。
- 软件开发环境/开发工具：Android Studio（Giraffe/Koala 及以上），JDK 17，Gradle（随工程 wrapper），Git。
- 该软件的运行平台/操作系统：Android 8.0 及以上；需网络、摄像头、麦克风、存储等权限支持。
